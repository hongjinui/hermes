Hermes 예외처리 공통화 작업을 수행합니다. `$ARGUMENTS`에 따라 아래 작업을 실행하세요.

## 인수가 없거나 `--audit`인 경우 — 전체 점검 후 보고서만 출력 (코드 수정 없음)

모든 `.py` 파일을 읽어 아래 항목을 심각도별로 분류해 보고서를 출력합니다.

### 심각도 기준
- **CRITICAL**: 프로그램 중단 가능성 (미처리 예외, 인덱스 직접 접근, DB/API 호출 무방비)
- **WARNING**: 데이터 유실 가능성 (저장 실패 무시, 로그 없는 except, 빈 except)
- **INFO**: 일관성 개선 권장 (로깅 포맷 불일치, 중복 패턴)

### 점검 항목
1. **try/except 누락**
   - 외부 리소스 접근 (DB, API, 파일 I/O, 네트워크) 중 try/except 없는 곳
   - `list[0]`, `dict["key"]` 직접 접근 (IndexError / KeyError 위험)

2. **불완전한 예외처리**
   - `except Exception: pass` — 예외 삼킴
   - `except Exception as e:` 뒤에 로그 없는 경우
   - 예외 잡고 잘못된 기본값 반환 (예: 실패해도 True 반환)

3. **로깅 불일치**
   - `print()` 사용 (logger 대신)
   - 로그 포맷이 `[모듈명]`, `(모듈명)` 등으로 혼재
   - ERROR/WARNING 레벨 혼용 기준 없음

4. **공통화 가능한 패턴**
   - 2개 이상 파일에서 반복되는 동일한 try/except 블록
   - utils.py로 추출하면 좋을 에러 처리 로직

### 보고서 형식
```
## CRITICAL
- [파일명:함수명] 설명
  현재 코드: ...
  위험: ...

## WARNING
- [파일명:함수명] 설명

## INFO
- [파일명:함수명] 설명

## 공통화 후보
- 패턴: ...
  위치: 파일A:줄, 파일B:줄
  제안: utils.py에 추출
```

---

## `--fix`인 경우 — 점검 후 수정까지 실행

1. 먼저 `--audit`과 동일하게 전체 점검
2. CRITICAL 항목부터 순서대로 수정
3. 각 수정 전 "어떤 파일의 어떤 함수를 왜 수정하는지" 설명 후 진행
4. 수정 후 영향받는 다른 파일(호출부) 확인
5. utils.py 공통 함수 추출이 필요하면 추가

수정 우선순위:
1. CRITICAL — 즉시 수정
2. 공통화 후보 — utils.py 추출
3. WARNING — 보완
4. INFO — 선택적

---

## `--report`인 경우 — 현재 예외처리 현황 요약표만 출력

각 파일별로 표 형태로 출력:

| 파일 | 함수 수 | try/except 수 | 누락 의심 | 공통화 후보 |
|------|--------|--------------|---------|-----------|

---

## 특정 파일명이 인수인 경우

해당 파일만 점검합니다. 예: `/error-handler database.py`
수정까지 원하면 `/error-handler database.py --fix`

---

## 주의사항
- 수정 시 기존 로직 변경 없이 예외처리만 추가/개선
- except 블록에서 반환값이 바뀌면 호출부 영향 확인 필수
- utils.py 추출 시 기존 import 누락 없도록 확인
